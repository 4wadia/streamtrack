# Specs Directory

This folder intentionally contains only operational docs that are still useful.

## Keep

- `IMPLEMENTATION_PLAN.md`: phase tracking and current delivery status.
- `API_TESTING.md`: endpoint reference and test-oriented request/response examples.
- `BUILD.md`: setup and run instructions.

## Removed

The removed docs were duplicates, outdated concept notes, or non-project artifacts that conflicted with current implementation.

## Update Rule

When functionality changes, update only the relevant retained spec file instead of creating a new overlapping doc.
